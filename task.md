### 2. Remove conflicting static files in `public/`

**Problem:** Next.js serves files in `public/` verbatim. Because `public/robots.txt`
and `public/sitemap.xml` exist, they take precedence over the files generated by
`app/robots.ts` and `app/sitemap.ts`. The static `sitemap.xml` is already stale
(only ~68 URLs, missing blog posts and brand slugs) and will never auto-update.

**Implementation steps:**
1. Delete `public/robots.txt`:
   ```bash
   rm public/robots.txt
   ```
2. Delete `public/sitemap.xml`:
   ```bash
   rm public/sitemap.xml
   ```
3. Run `npm run build` — verify `/robots.txt` and `/sitemap.xml` still appear
   in the route manifest (they are generated by `app/robots.ts` and `app/sitemap.ts`).
4. Commit the deletions.

> **Note:** If the project is deployed to a static host that does not run Next.js
> server (e.g. plain Apache/XAMPP), keep the static files but regenerate them
> after every build by copying `.next/server/app/sitemap.xml` and
> `.next/server/app/robots.txt` to `public/`.

### 3. `components/NewsletterForm.tsx` — Wire up real submit action

**Problem:** `onSubmit={(e) => e.preventDefault()}` means the form does nothing.
Emails typed by users are silently discarded.

**Option A — Resend API (recommended, free tier available):**
1. Install: `npm install resend`
2. Create `app/api/newsletter/route.ts`:
   ```ts
   import { Resend } from 'resend';
   import { NextResponse } from 'next/server';

   const resend = new Resend(process.env.RESEND_API_KEY);

   export async function POST(req: Request) {
     const { email } = await req.json();
     await resend.emails.send({
       from: 'newsletter@vedicheritage.in',
       to: 'your@email.com',
       subject: 'New subscriber',
       text: `New subscriber: ${email}`,
     });
     return NextResponse.json({ ok: true });
   }
   ```
3. Add `RESEND_API_KEY=re_xxxx` to `.env.local`.
4. In `NewsletterForm.tsx`, replace the handler:
   ```ts
   const [status, setStatus] = useState<'idle'|'loading'|'done'|'error'>('idle');
   const handleSubmit = async (e: React.FormEvent<HTMLFormElement>) => {
     e.preventDefault();
     setStatus('loading');
     const email = new FormData(e.currentTarget).get('email');
     const res = await fetch('/api/newsletter', {
       method: 'POST',
       body: JSON.stringify({ email }),
       headers: { 'Content-Type': 'application/json' },
     });
     setStatus(res.ok ? 'done' : 'error');
   };
   ```
5. Show a success message when `status === 'done'`.

**Option B — WhatsApp fallback (no backend required):**
Replace the submit handler with:
```ts
const handleSubmit = (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
  const email = new FormData(e.currentTarget).get('email')?.toString() ?? '';
  const text = encodeURIComponent(`Newsletter signup: ${email}`);
  window.open(`https://wa.me/919999999999?text=${text}`, '_blank');
};
```
